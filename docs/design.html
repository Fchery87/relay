Do's and Don'ts

Guardrails that keep the switchboard useful rather than theatrical.

Do

  • ✓Keep the active thread, run stage, model, permissions, branch, and pending approvals visible near the work they affect.
  • ✓Use the handoff trace only when order or propagation clarifies the run.
  • ✓Make safe defaults, keyboard focus, hover, pressed, disabled, loading, and destructive states unambiguous.
  • ✓Reuse the same primitives across workbench, orchestration, setup, and administration.
  • ✓Let terminal output, diffs, logs, and agent events use the full available working area.
  • ✓Reveal secondary surfaces contextually and preserve the user's density preference.

Don't

  • ×Add gradients, glows, decorative telemetry, KPI-card grids, or status theatrics.
  • ×Turn the handoff trace into a looping animation, progress decoration, or false precision.
  • ×Use chat bubbles, oversized welcome panels, ornamental copy, or excessive whitespace.
  • ×Crowd the shell with permanent toolbars, unlabeled icons, or every tool at once.
  • ×Use color alone to convey status, permission, or risk.
  • ×Round structural panes, turn routine actions into pills, or hide agent operations behind friendly but vague abstractions.
